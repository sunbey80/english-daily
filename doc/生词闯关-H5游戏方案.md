# 生词闯关 —— 基于生词本的 H5 复习小游戏

> 目标：把用户生词本里的词放进**短时高频、主动回忆**的闯关里反复接触，达到吸收新词的目的。
> 核心差异：① 在词**原本出现的故事句子**里复习（情境编码）；② 和阅读**共用同一套 SRS 词汇画像**——游戏里的答题回写 `user_word` 的 `due_at`，影响明天章节是否再让这个词复现。游戏不是孤立小玩具，而是护城河的一部分。

---

## 1. 设计原则

- **主动回忆 > 被动认读**：让用户拼出/选出，而非只看一眼（测试效应）
- **间隔重复**：优先考 `due_at <= 今天` 的到期词
- **情境编码**：在词原句里复习，比孤立卡片记得牢
- **多模态**：词形 + 发音(TTS) + 中文，多通道编码
- **短时高频**：单局 60–90 秒，即时反馈、连击、连胜

## 2. MVP 玩法：「每日 60 秒生词闯关」

一局抽 **8–12 个到期生词**，每词一道题，混合题型，限时连击：

| 题型 | 形式 | 练什么 |
|---|---|---|
| **情境填空**（主） | 该词原句英文挖空 + 整句中文 + 首字母提示，用户拼出 | 情境 + 主动回忆 |
| **看中选英** | 给中文释义，从 4 个英文词里选 | 认读 |
| **听音抓词** | TTS 念词，从 4 个里选 / 拼 | 听力通道 |

- **难度随熟练度递进**（和 FSRS stability 挂钩）：生疏→选择；较熟→填空给首字母；很熟→纯拼写。MVP 先按"有原句→填空，否则→选择"分配。
- **结算**：本局得分、最高连击、复习词数、连胜天数。

## 3. 数据来源（零新增 API 成本）

| 需要 | 来源 |
|---|---|
| 到期生词 | `user_word`(state=1) 按 `due_at` 升序 + `word`(lemma/zh_gloss/phonetic_us) |
| 原句 + 中文 | 在所有章节 `chapter.translation`(jsonb) 里搜含该词的句子 |
| 干扰项 | 本地从 `word` 表取同类词（无需 LLM） |
| 发音 | 浏览器 `speechSynthesis`（免费、客户端） |

> 说明：当前 schema 未记录"某生词来自哪句"，MVP 先用"全章节句子里搜含该词的句子"兜底；后续可在查词时记录来源句以更精准。

## 4. 答题回写（接入 SRS，加深护城河）

每题作答 → 视为一次复习事件，喂给 FSRS（`lib/fsrs.ts` 的 `rate`）：
- 答对 → `good`：`due_at` 推远、stability 升
- 答错 → `again`：`due_at` 拉近、stability 降
- `exposures += 1`；达阈值且稳定 → `state=2` 退出复现池

→ 游戏强化的是和阅读**同一份** `user_word`，复习过的词在明日章节更不易再现，形成"读+玩"统一的词汇画像。

## 5. 技术结构

```
app/play/page.tsx          # 入口（需登录）
app/play/PlayClient.tsx    # 客户端游戏 UI（计时/连击/题目交互/TTS）
app/api/play/session/route.ts   # GET：发一局题目
app/api/play/answer/route.ts    # POST：提交单题结果 → FSRS 回写
lib/play.ts                # 组局：取到期词 + 找原句 + 生成干扰项 + 定题型
```

- 鉴权：复用 `getRequestUser`（生词本是 per-user）。未登录引导去登录。
- 取题接口返回：`[{ word_id, lemma, zh_gloss, phonetic_us, type, sentence?{en,zh}, options?[], answer }]`
- 提交接口入参：`{ word_id, correct }` → 读 `user_word` → `rate` → upsert。
